OpenCLI 中文教程：把网站和桌面 App 直接变成命令行

一句话判断：OpenCLI 的价值，不是又一个自动化工具，而是把你已经在用的网站和 Electron App 直接变成可脚本化、可复用、可让 AI 调用的 CLI 接口。

最可能给我们提速的 4 件事：
  1. 复用 Chrome 登录态：不用自己折腾登录、Cookie、反复扫码，直接吃浏览器里现成会话。
  2. 把桌面工具 CLI 化：Cursor、Codex、Antigravity、ChatGPT 这类 Electron App，可以直接被命令行和工作流调度。
  3. 用 `explore / synthesize / generate` 快速做新适配器：碰到新网站时，不必从零硬写脚本。
  4. 用 YAML / TypeScript 动态注册命令：高频站点和固定流程可以沉淀成自己的 CLI 资产。

一、OpenCLI 到底是什么

OpenCLI 是一个把网站或 Electron App 变成命令行接口的工具。它同时覆盖两类场景：

  • 浏览器侧：Twitter/X、小红书、知乎、B站、YouTube、雪球、LinkedIn 等。
  • 桌面 App 侧：Cursor、Codex、ChatGPT、ChatWise、Discord、Notion、Antigravity 等。

它最适合的不是“一次性录宏”，而是把你已经在用的网页和桌面工具，重新整理成能被脚本、Agent 和工作流直接调用的执行入口。

二、为什么这条对我们有现实意义

能力 对我们的现实意义
复用浏览器登录态 少掉一大截登录、鉴权、Cookie 搬运和反复排错成本。
控制桌面 App 把 Codex、Cursor、ChatGPT 这类原本只能手点的工具拉进自动化链路。
AI-native 探索链路 遇到新站点时，更快找到可用接口、页面能力和认证策略。
动态加载适配器 适合把自己常用站点和固定动作沉淀成团队长期资产。

三、简洁教程：最短上手步骤

  1. 确保 Node.js 版本至少是 20。
  2. 安装：npm install -g @jackwener/opencli
  3. 给 Chrome 装 Browser Bridge 扩展。
  4. 在 Chrome 里提前登录你要操作的网站。
  5. 跑 opencli doctor --live 检查扩展、daemon 和浏览器连通性。
  6. 跑 opencli list 看当前可用命令。
npm install -g @jackwener/opencli
opencli doctor --live
opencli list
opencli twitter trending
opencli xiaohongshu search --keyword "AI Agent"
opencli cursor status
opencli codex status

四、先只记住这 5 个命令

1. 看所有命令：opencli list

2. 查扩展和浏览器连通性：opencli doctor --live

3. 直接跑现成站点命令：opencli twitter search --query "AI Agent"

4. 控制桌面 App：opencli codex status / opencli cursor status

5. 给新网站快速起适配器：opencli generate https://example.com --goal "hot"

五、圈出来：哪些最值得我们优先试

5.1 Chrome 登录态复用

这是最现实、最立竿见影的提速点。很多自动化项目死在登录、鉴权、会话保持和风控处理上，而 OpenCLI 直接走浏览器已登录会话，天然就省掉一大截麻烦。

5.2 Electron App CLI 化

对我们这种会同时用 Codex、Cursor、ChatGPT、Antigravity 的场景，这点最有想象力。它意味着这些桌面工具不再只是 GUI，而是可以进入更大的自动化系统。

5.3 `explore / synthesize / generate`

对新网站，最慢的通常是“先理解页面、再猜接口、再试认证”。OpenCLI 的探索链路就是在压缩这段时间。哪怕最后还需要手工修 adapter，也比完全从零摸索快很多。

5.4 下载与导出能力

小红书、Twitter、B站、知乎这类平台，如果你本来就有素材采集、导出文章、下载媒体的需求，OpenCLI 的内置下载支持会比你自己每个平台重写一遍更省事。

六、适合我们先试的场景

  • 内容增长：抓 Twitter/X、小红书、知乎、B站内容，做素材池和二次分发。
  • 研发提效：先用 `cursor / codex / antigravity` 做状态检查、读历史、导出结果。
  • 资料采集：知乎文章导出、Twitter 媒体下载、B站视频下载、小红书素材下载。
  • 新站点接入：碰到一个新网站，先用 `explore` 和 `generate` 试水，别急着自己手搓整套脚本。

七、使用时要注意什么

  • 它依赖你本机 Chrome 已经登录目标站点。
  • Browser Bridge 扩展和本地 daemon 是前提，建议先跑 `doctor`。
  • 桌面 App 适配器很强，但不同系统和 App 版本可能会有差异，别一上来就赌全自动。
  • 最好的做法不是全盘替换现有工作流，而是先挑一两个高频场景试。
一句话建议：如果你最近最想提速的是“复用网页登录态 + 接管桌面 AI 工具 + 给新站点快速起自动化入口”，OpenCLI 很值得先试；如果你只是想做一次性网页点击脚本，它的价值反而没那么大。
