OpenClaw 量化交易的实际应用 1：MiniQMT 自动交易

说话就能做量化。从本地部署、飞书接入到 `miniqmt-xttrade` Skills，这篇把“查询账户 -> 下单 -> 撤单”的最短闭环先打通。

改编自 微信原文

这篇实战的核心不是“再教你写一套交易框架”，而是把 OpenClaw 变成真正能操作本地 MiniQMT 的执行 Agent。你用自然语言说出交易意图，它负责理解、拼参数、调用技能、落到本机交易终端执行。

一、为什么要做“说话做量化”

  • 量化想法很多，但一碰到 Python、环境变量、接口文档和状态同步就被卡住。
  • 传统自动下单链路长，要啃文档、写回调、处理异常和订单状态。
  • 策略多数跑在本地机器或云服务器上，人在外面时难以及时干预。

OpenClaw 的价值在这里很直接：它不只是聊天，而是能操作本地电脑的 Agent。只要 MiniQMT 和 OpenClaw 在同一台机器上，很多原本需要手写的底层动作都可以压缩成自然语言指令。

二、这一套先打通什么能力

基础交互

对话式查账户、查持仓、查委托。

交易执行

自然语言买入、卖出、限价、市价、撤单。

移动干预

接入飞书后，通勤路上也能下指令。

三、本地部署要点

最关键的一条：OpenClaw 必须部署在运行 MiniQMT 的本地电脑上。因为只有在同一台机器里，它才有权限调用你的本地交易终端。

推荐最短路径：

  1. Windows PowerShell 一键安装 OpenClaw：iwr -useb https://openclaw.ai/install.ps1 | iex
  2. 执行 openclaw onboard 完成基础引导。
  3. 接入一个可用大模型，作为 OpenClaw 的“大脑”。
  4. 接入飞书，方便在手机端随时发指令。
  5. 安装 `miniqmt-xttrade` Skills 与常用技能。
  6. 确保 MiniQMT 已登录，并且使用的是 MiniQMT 版本。

四、交易核心：miniqmt-xttrade Skills

真正让大模型能“安全且精确”操作股票账户的关键，不是提示词，而是技能层。`miniqmt-xttrade` 本质上是一套封装好的原生交易指令集，放进 OpenClaw 的 `skills/` 目录后，大模型才看得懂账户查询、报单、撤单和银证转账这些动作。

  • query_account.py：查可用资金、持仓市值、总资产。
  • query_position.py：查当前持仓与浮盈浮亏。
  • query_order.py / cancel_order.py：查委托、查未成单、执行撤单。
  • order_stock.py：支持买卖、限价、市价、最新价和多种报价模式。
  • bank_transfer.py：支持银证转账。

比如你说“帮我全仓市价买入浦发银行”，OpenClaw 会自动拆解出代码、方向、价格类型和数量，再拼成底层脚本调用参数，完成本地执行。

五、这篇演示的闭环

场景 1：查账户
“帮我查下当前股票账户里还有多少可用资金？”

场景 2：精准下单
“马上帮我以 10.5 元的价格限价买入 100 股 600000.SH。”

场景 3：撤单
“慢着，帮我把刚才那笔 600000.SH 的买单撤销掉。”

重点不是单个命令，而是这条链路已经通了：查询账户 -> 提交委托 -> 找回订单 -> 发起撤单。这说明 OpenClaw 在量化场景里不只是“能分析”，而是真的可以接管执行。

六、后续可以继续扩展什么

  • 自然语言条件选股：连续资金净流入、低市盈率、AI 概念等。
  • 新闻情绪分析后自动调仓。
  • 对话式回测并把收益曲线发到飞书或微信。
  • 长财报扫描，自动提取风险点和业绩亮点。
