🔎 搜索配置

决定 Agent 会不会查资料
关键配置 Web Search

OpenClaw 的 Web Search 为什么这么关键

讨论结论很明确：搜索配置会显著影响 Agent 查资料、核实信息和跟踪时效信息的能力。配置对了，OpenClaw 才更像“会搜、会判、会引用”的执行代理。

默认优先：有卡且追求兼容性，优先 Brave Search API。

零门槛优先：不想绑卡，优先 Tavily。

进一步降成本：Exa、You.com、OpenRouter + Perplexity、多引擎方案都值得试。

备注：这里的免费额度是社区讨论中的常见说法，适合做选型参考，不建议把配额当长期承诺。

搜索 API 免费优先

最值得先试的几条搜索路线

给 OpenClaw 配搜索，不是比“谁更全”，而是比“谁更适合 Agent”。下面这几条路线各自适合不同约束。

Brave：社区常提每月约 2000 次免费，速度快，结构化结果好，但通常要绑卡。

Tavily：社区常提每月约 1000 次免费，不绑卡，对 Agent 友好，开箱即用。

Exa / You.com：更偏语义搜索和赠送额度思路，适合继续压低成本。

multi-search-engine skill：适合把多个搜索源拼起来做容灾和补盲。
